# 义务维修活动报名系统 - 设计文档

## 概述

为大学计算机协会每周三义务维修活动构建的手机端报名与数据管理系统。参与者无需登录即可报名，管理员通过固定密码查看数据、可视化分析和导出。

## 技术方案

- **前端**：Vue 3 + Vite + Element Plus（移动端友好组件库）
- **后端**：Python FastAPI（异步框架，自动生成 API 文档）
- **数据库**：SQLite（零配置，30 人规模完全够用）
- **图表**：ECharts（中文友好图表库）
- **导出**：openpyxl（Excel 生成）

## 功能需求

### 1. 参与者报名页（`/`）

- 显示最近 4 周的周三日期列表
- 每个日期下显示两个班次：13:30-15:30、15:30-17:30
- 每个班次最多 5 人，满员后按钮变灰不可点击
- 每个班次显示已报名人数（如 3/5）
- 最新一周突出显示为"本周活动"，前几周折叠为"历史活动"（可展开查看）
- 点击班次后弹出输入姓名的表单
- 提交后实时更新报名人数
- 同一天同一班次不允许同名重复报名

### 2. 管理后台（`/admin`）

- 固定密码登录（环境变量配置）
- **数据总览**：4 个统计卡片
  - 本周报名人数
  - 累计参与人次
  - 第一班人数
  - 第二班人数
- **每周趋势图**：柱状图对比两个班次的每周报名人数
- **个人排行榜**：按累计参与次数排名
- **报名列表**：支持按日期、班次筛选，每条记录可删除
- **导出 Excel**：按日期范围导出报名数据

## 数据模型

### activities 表

| 字段 | 类型 | 说明 |
|------|------|------|
| id | INTEGER PRIMARY KEY | 自增主键 |
| date | TEXT NOT NULL | 活动日期，格式 YYYY-MM-DD |
| shift | TEXT NOT NULL | 班次：`13:30-15:30` 或 `15:30-17:30` |

### signups 表

| 字段 | 类型 | 说明 |
|------|------|------|
| id | INTEGER PRIMARY KEY | 自增主键 |
| name | TEXT NOT NULL | 参与者姓名 |
| activity_id | INTEGER NOT NULL | 关联活动 ID（外键） |
| created_at | TIMESTAMP | 报名时间，默认当前时间 |

**约束**：同一 activity_id 下 name 唯一（防止重复报名）

## API 设计

### 参与者接口

#### GET /api/activities

获取可用的活动列表（最近几周的日期+班次，含已报名人数）。

**响应**：
```json
[
  {
    "id": 1,
    "date": "2026-05-07",
    "shift": "13:30-15:30",
    "signup_count": 3,
    "max_signup": 5,
    "is_full": false
  }
]
```

#### POST /api/signups

提交报名。

**请求**：
```json
{
  "name": "张三",
  "activity_id": 1
}
```

**响应**：
- 201 Created：报名成功
- 409 Conflict：重复报名
- 400 Bad Request：该班次已满（5/5）

### 管理员接口（需 Authorization: Bearer <password>）

#### POST /api/admin/login

验证密码，返回 token。

#### GET /api/admin/signups

获取报名列表，支持 `date` 和 `shift` 查询参数筛选。

#### GET /api/admin/stats

获取统计数据，返回本周报名数、累计参与人次、每周趋势数据、个人排行榜。

#### DELETE /api/admin/signups/{id}

删除指定报名记录。

#### GET /api/admin/export

导出 Excel 文件，支持 `start_date` 和 `end_date` 查询参数。

## 页面路由

| 路径 | 页面 | 说明 |
|------|------|------|
| `/` | 报名页 | 参与者使用，无需登录 |
| `/admin` | 管理后台 | 密码保护，含数据总览、图表、列表、导出 |

## UI 设计

### 报名页（列表式布局）

- 顶部标题：义务维修报名 + 副标题
- **本周活动**：突出显示，两个班次卡片，每个显示 x/5 人数、报名按钮
  - 满员（5/5）时按钮变灰，文字显示"已满"
- **历史活动**：折叠区域，点击展开查看前几周的报名情况（只读）
- 点击报名按钮弹出姓名输入弹窗
- 配色：蓝色主色调（#1890ff），白底卡片，灰色背景

### 管理后台

- 顶部标题 + 退出登录按钮
- Tab 切换：数据总览 / 报名列表 / 排行榜
- 数据总览：4 个统计卡片 + 柱状趋势图
- 报名列表：日期/班次筛选 + 列表 + 删除 + 导出按钮
- 排行榜：按参与次数排序的姓名列表

## 部署

- 后端 FastAPI 服务，前端打包为静态文件由 FastAPI 托管
- SQLite 数据库文件存储在项目目录
- 管理员密码通过环境变量 `ADMIN_PASSWORD` 配置

## 活动日期生成逻辑

- 后端动态计算：从本周三开始，往前推 4 周，共生成 8 条活动记录（4 周 × 2 班次）
- 不在数据库中预存活动记录，每次请求 `/api/activities` 时动态生成日期列表
- 已报名人数通过关联查询 signups 表获取
- 如果某周的活动记录尚无报名，也会展示（方便提前报名）

## 约束与假设

- 使用人数 30 人以内，无需高并发优化
- 不需要用户注册登录，报名仅需姓名
- 管理员仅一人，使用固定密码
- 每周三固定举办，不考虑节假日停办的情况（管理员可手动删除误报名数据）
